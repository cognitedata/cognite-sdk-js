FeedbackStatus

FeedbackStatus: "CREATED" | "ACCEPTED" | "REJECTED" | "STALE"
  • Defined in packages/playground/src/api/documents/types.gen.ts:903

Status of the feedback, complying with the lifecycle described below.

Feedback lifecycle
  • It is initially CREATED.
  • If the administrator accepts the feedback, it moves to ACCEPTED. Once in this state, it cannot move to another state.
  • If the administrator rejects the feedback, it moves to REJECTED. Once in this state, it cannot move to another state.
  • If the feedback object has become invalid, it moves to STALE. Once in this state, it cannot move to another state.

Definition of feedback invalidity
  • If the action was ATTACH, and the label to attach does not exist anymore.
  • If the action was ATTACH, and the label is already attached to the file.
  • If the action was DETACH, and the label is not attached to the file anymore.
example

ACCEPTED
