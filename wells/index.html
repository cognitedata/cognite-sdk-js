Cognite JavaScript SDK

Cognite Wells JS SDK (derived from stable)

This package provides an SDK derived from @cognite/sdk, aka stable.

It is recomended to install this package under the same name as @cognite/sdk. This allows you to change SDK versions without changing your imports. See the beta readme for details.

install and build

 
yarn install
yarn build

consuming
set your env variables (must be valid for both cdf and geospatial API)
 
COGNITE_WELLS_PROJECT=<project-tenant>
COGNITE_WELLS_CREDENTIALS=<your-api-key>
setup client
 
let client: CogniteClient = setupLoggedInClient()
run a query
 
    const polygon = <GeoJson>{
      type: 'Polygon',
      coordinates: [
        [
          [-4.86423, 63.59999],
          [19.86423, 63.59999],
          [19.86423, 52.59999],
          [-4.86423, 52.59999],
          [-4.86423, 63.59999],
        ],
      ],
    };

    const response = await client.wells.getWellsByPolygon({geometry: polygon});

Testing the wells package only

This repo contains some integration tests that require a CDF api key for subsurface-test tenant. Talk to any of the contributors or leave an issue and it'll get sorted. Travis will run the test and has its own api key.

Run all tests:

navigate to wells package root directory:

cd /cognite-sdk-js/packages/wells
yarn build
COGNITE_WELLS_PROJECT=<project-tenant> COGNITE_WELLS_CREDENTIALS=<your-api-key> yarn test
